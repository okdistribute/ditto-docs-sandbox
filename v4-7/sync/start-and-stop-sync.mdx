---
title : Starting and Stopping Sync
description : "Required only once, initiating the sync process automatically connects you with the mesh network. Once connected, you immediately begin receiving updates from set sync subscriptions and sending updates to subscribing remote peers."
---

## Starting Sync

To initiate the sync process:

<CodeGroup>

```swift Swift

try! ditto.startSync()
```

```kotlin Kotlin
try {
    ditto.startSync()
} catch (e: DittoError) {
    // handle error
}
```

```javascript JS
try {
  ditto.startSync()
} catch (err) {
  // handle error
}
```

```java Java
try {
    ditto.startSync();
} catch(DittoError e) {
    // handle error
}
```

```csharp C#
try
{
    ditto.StartSync();
}
catch (DittoException ex)
{
    // handle exception
}
```

```cpp C++
try {
  ditto.start_sync();
} catch (const DittoError &err) {
  std::cerr << err.what();
}
```

```rust Rust
ditto.try_start_sync()?;
```

```dart Dart
try {
  await ditto.startSync();
} catch (e) {
  // handle exception
}
```

</CodeGroup>

<Info>
Ensure the sync process remains active throughout your app's lifecycle by starting the sync process (`startSync`) in the top-most scope of your code.
</Info>

## Stopping Sync

To stop the sync process, call the `stopSync` function. Once called, you will be disconnect from any peers and all active sync subscriptions will be paused.

<CodeGroup>

```swift Swift

try! ditto.stopSync()
```

```kotlin Kotlin
try {
    ditto.stopSync()
} catch (e: DittoError) {
    // handle error
}
```

```javascript JS
try {
  ditto.stopSync()
} catch (err) {
  // handle error
}
```

```java Java
try {
    ditto.stopSync();
} catch(DittoError e) {
    // handle error
}
```

```csharp C#
try
{
    ditto.StopSync();
}
catch (DittoException ex)
{
    // handle exception
}
```

```cpp C++
try {
  ditto.stop_sync();
} catch (const DittoError &err) {
  std::cerr << err.what();
}
```

```rust Rust
ditto.try_stop_sync()?;
```

```dart Dart
ditto.stopSync();
```

</CodeGroup>
